# LoadOut – Setup

Flutter + Firebase ammo reloading app, targeting iOS and Android.

## Identifiers

| | |
|---|---|
| App name | LoadOut |
| Store name | LoadOut: Precision Reloading |
| Bundle ID / package | `com.johnsondigital.loadout` |
| Firebase project ID | `loadout-precision-reloading` |
| Firestore region | `nam5` |
| Firebase console | https://console.firebase.google.com/project/loadout-precision-reloading |

## Already done

- Firebase project created (`loadout-precision-reloading`)
- iOS and Android apps registered under bundle `com.johnsondigital.loadout`
- `lib/firebase_options.dart`, `ios/Runner/GoogleService-Info.plist`, and
  `android/app/google-services.json` generated by `flutterfire configure`
- Cloud Firestore API enabled, default database provisioned in `nam5` (Native mode)
- Security rules in `firestore.rules` deployed (each user only sees their own
  data under `users/{uid}/...`)

## One remaining manual step — enable Email/Password auth

Auth providers can't be toggled from the CLI. Open the Firebase Console once:

1. https://console.firebase.google.com/project/loadout-precision-reloading/authentication/providers
2. Click **Get started** if prompted, then enable **Email/Password**
3. Save

## Run

```sh
flutter pub get
flutter run        # iOS or Android (first iOS run will trigger pod install)
```

## Day-to-day commands

```sh
# Update Firestore rules after editing firestore.rules
firebase deploy --only firestore:rules --project=loadout-precision-reloading

# Re-sync Firebase configs (e.g. after adding a platform)
flutterfire configure --project=loadout-precision-reloading

# Static analysis
flutter analyze
```

## Project layout

```
lib/
  main.dart                  Firebase init + runApp
  app.dart                   Root widget, providers, auth gate
  firebase_options.dart      Generated by flutterfire configure
  models/                    Plain data classes (Load, Bullet, Powder, ...)
  services/                  AuthService, FirestoreService
  repositories/              Stream-based access to per-user collections
  screens/                   UI by feature (auth, home, loads, components)
  theme/                     App theme
  utils/                     (reserved)
  widgets/                   (reserved – shared widgets)
```

Firestore data is namespaced under `users/{uid}/...` so each account's loads
and components stay private — see `firestore.rules`.
